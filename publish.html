Package Publishing
In order to be able to publish a package, it needs to follow these rules in order for it to be safe to use by others:

- The package must NOT be harmful: anyone who dares to put the smallest bit of malware, backdoor, etc. will be blocked from submitting any packages to the repository.

- Remember, the package must be tested and working properly: nobody wants a bad or broken package that will making their dpkg go boom boom shakalaka.

- No bootloopers: I mean come on. If you guys are going to make bootlooping tweaks then don't host it on this repo. Host it on your personal one.

- No pirating other tweaks: Developers put very hard work into their official tweaks until they realize pirated versions are hosted. All I'm asking is to just don't do it on a public repo.
Publish a package
Currently there is no way of publishing via the website so you will have to email at [INTOJBPUBLISH_EMAIL] with these pieces of information:

- Name of the package
- Description
- Author name and email
- Section the package belongs to (Tweaks, Terminal Support, Themes, Applications, etc.)
- Version number

After that just send the email and it will be reviewed shortly.
